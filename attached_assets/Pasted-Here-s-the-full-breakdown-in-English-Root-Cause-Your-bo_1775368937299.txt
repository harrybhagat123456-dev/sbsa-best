## 🔍 Root Cause

Your bot (`drm_handler.py`) crashes when it receives `'001-002'` because it tries to do `int('001-002')` — Python can't convert a range string to an integer.

The error happens at **3 places** in your `modules/drm_handler.py`:
- **Line ~112:** `if int(raw_text) > len(links):`
- **Line ~213:** `count = int(raw_text)`
- **Line ~550:** `success_count = len(links) - int(raw_text) - failed_count + 1`

## 📊 Why UDAY works but Sankalp doesn't

| | Sankalp file ❌ | UDAY file ✅ |
|---|---|---|
| **Format** | `(Category) Title: //url` | `Title://url` |
| **Subject prefix** | `(Ethics, Integrity and Aptitude) Ethics 01:` | `Ethics 01:` |
| **Extra text** | Has `No DMP:`, `Rescheduled @06:00 PM:` etc. | Cleaner, no extra metadata |

The `()` parentheses in the Sankalp file confuse the bot's parser. When the bot asks the user for a number (like "resume from which point?"), it receives something like `001-002` from the misparsed text instead of a clean integer.

### Step 1: Add this helper function at the top of the file (after imports):

```python
def parse_number_or_range(raw_text):
    """Parse '5', '001-002', '1-10' etc. Returns (start, end) tuple or None."""
    raw_text = raw_text.strip()
    if '-' in raw_text and not raw_text.startswith('-'):
        parts = raw_text.split('-')
        if len(parts) == 2:
            try:
                start = int(parts[0].strip())
                end = int(parts[1].strip())
                return (start, end)
            except ValueError:
                pass
    try:
        return (int(raw_text), int(raw_text))
    except ValueError:
        return None
```
